cassandra:start

Full name:

org.codehaus.mojo:cassandra-maven-plugin:2.1.7-1:start

Description:

Starts a Cassandra instance in the background.

Attributes:

  • Requires a Maven project to be executed.
  • The goal is thread-safe and supports parallel builds.
  • Binds by default to the lifecycle phase: pre-integration-test.

Required Parameters

Name Type Since Description
cassandraDir File - The directory to hold cassandra's database.
Default value is: ${project.build.directory}/cassandra.
classesDirectory File - The directory containing generated classes.
User property is: project.build.outputDirectory.
testClassesDirectory File - The directory containing generated test classes.
User property is: project.build.testOutputDirectory.

Optional Parameters

Name Type Since Description
addMainClasspath boolean - Adds the main classpath to cassandra (for example you could use this when you have a custom comparator on your main classpath.
Default value is: false.
addTestClasspath boolean - Adds the test classpath to cassandra (for example you could use this when you have a custom comparator on your test classpath.
Default value is: false.
cqlVersion String 1.2.1-2 Version of CQL to use
User property is: cql.version.
cuDataSet File 1.2.1-2 The CassandraUnit dataSet to load.
Default value is: ${basedir}/src/test/resources/dataSet.xml.
cuLoadAfterFirstStart boolean 1.2.1-2 When true, if this is a clean start then the CassandraUnit dataSet will be applied automatically.
Default value is: true.
User property is: cassandra.cu.load.after.first.start.
cuLoadFailureIgnore boolean 1.2.1-2 Whether to ignore errors when loading the script.
User property is: cassandra.cu.load.failure.ignore.
jmxPort int - Port to listen to for the JMX interface.
Default value is: 7199.
User property is: cassandra.jmxPort.
keyspace String - The keyspace against which individual operations will be executed
User property is: cassandra.keyspace.
listenAddress String - Address to bind to and tell other Cassandra nodes to connect to. You must change this if you want multiple nodes to be able to communicate! Leaving it blank leaves it up to InetAddress.getLocalHost(). This will always do the Right Thing if the node is properly configured (hostname, name resolution, etc), and the Right Thing is to use the address associated with the hostname (it might not be). Setting this to 0.0.0.0 is always wrong. Do not change this unless you really know what you are doing.
Default value is: 127.0.0.1.
loadAfterFirstStart boolean - When true, if this is a clean start then the load script will be applied automatically.
Default value is: true.
User property is: cassandra.load.after.first.start.
loadFailureIgnore boolean - Whether to ignore errors when loading the script.
User property is: cassandra.load.failure.ignore.
maxMemory int - Number of megabytes to limit the cassandra JVM to.
Default value is: 512.
User property is: cassandra.maxMemory.
nativeTransportPort int 2.0.0-1 Port on which the CQL native transport listens for clients.
Default value is: 9042.
User property is: cassandra.nativeTransportPort.
rpcAddress String - Address to use for the RPC interface. Do not change this unless you really know what you are doing.
Default value is: 127.0.0.1.
rpcPort int - Port to listen to for the RPC interface.
Default value is: 9160.
User property is: cassandra.rpcPort.
script File - The CQL file to load.
Default value is: ${basedir}/src/cassandra/cql/load.cql.
skip boolean - Skip the execution.
Default value is: false.
User property is: cassandra.skip.
startNativeTransport boolean 2.0.0-1 Enable or disable the native transport server. Currently, only the Thrift server is started by default because the native transport is considered beta.
Default value is: false.
User property is: cassandra.startNativeTransport.
startWaitSeconds int - How long to wait for Cassandra to be started before finishing the goal. A value of 0 will wait indefinitely. A value of -1 will not wait at all.
Default value is: 180.
stopKey String - Key to be provided when stopping cassandra
Default value is: cassandra-maven-plugin.
User property is: cassandra.stopKey.
stopPort int - Port to listen to for receiving the stop command over
Default value is: 8081.
User property is: cassandra.stopPort.
storagePort int - Port to listen to for the Storage interface.
Default value is: 7000.
User property is: cassandra.storagePort.
systemPropertyVariables Map 1.2.1-2 List of System properties to pass to the JUnit tests.
yaml String - In yaml format any overrides or additional configuration that you want to apply to the server.
